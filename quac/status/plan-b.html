Plan B · Status QUAC en la app legacy (app-qac)

Para qué sirve este documento: si la app v2 (app-quac-v2) no alcanza a salir a tiempo para el lanzamiento de Status QUAC, esta es la guía para que el dev del legacy implemente mínimo viable del programa de fidelización: cliente ve su nivel + tema de color por status. Audiencia: dev mobile del legacy (app-qac Flutter v3.7.0). Alcance del Plan B: SOLO renderizar el nivel del cliente con tema de color correspondiente. NO incluye pantalla "Mi nivel" completa, NO incluye comparar niveles, NO incluye mecánica de Quacs/puntos. Es la versión mínima para "estar en el aire" con Status QUAC mientras la v2 se completa.

Objetivo

Que el cliente vea su nivel actual reflejado en el tema de color de la app legacy.

Resultado visual esperado:

Nivel Header del Home Acento de botones
QUAC (base) Morado original #7633FF Morado
PLUS Gradiente vinotinto-morado Vinotinto
SILVER Gradiente plateado Gris oscuro
GOLD Gradiente dorado Dorado
BLACK Negro con acentos plateados (dark mode) Plateado claro

Endpoints disponibles (ya existen en el CRM Baguer)

El equipo CRM ya tiene los 3 endpoints listos en https://apibaguer.azurewebsites.net/api. No requieren desarrollo backend nuevo. La app legacy puede llamarlos directo desde el cliente (igual que ya hace con otros endpoints del CRM).

Campos clave para el Plan B:

  • categoria.categoria (int 1-5) → este es el único campo que necesitas para decidir el tema de color

Nota importante: el nit en la respuesta viene como 447624.0 (decimal). Conviértelo a int en el cliente.

Lo mínimo que debe hacer el legacy (Plan B mínimo viable)

1. Llamar al endpoint 3 al cargar el Home

Cuando el usuario hace login y el app levanta el Home, hacer en paralelo con get_profile:

text
GET https://apibaguer.azurewebsites.net/api/StatusQuac/cliente/{nit}

Y guardar el response en el provider que usen para el contexto del usuario (igual que se guarda cupoInfo).

2. Mapear categoria.categoria a tema de color

Tabla de mapeo que el dev tiene que implementar en el código mobile:

dart
// Pseudocódigo Flutter para el legacy
final Map<int, QuacTheme> statusThemes = {
  1: QuacTheme(
    headerGradient: LinearGradient(colors: [Color(0xFF7633FF), Color(0xFF5e26c9)]),
    accentColor: Color(0xFF7633FF),
    isDarkMode: false,
    label: 'QUAC',
  ),
  2: QuacTheme(
    headerGradient: LinearGradient(colors: [Color(0xFF8b2849), Color(0xFF5a1e6d), Color(0xFF2c0d6e)]),
    accentColor: Color(0xFF8b2849),
    isDarkMode: false,
    label: 'PLUS',
  ),
  3: QuacTheme(
    headerGradient: LinearGradient(colors: [Color(0xFFc8ccd2), Color(0xFFa3a8b1), Color(0xFF8e9299)]),
    accentColor: Color(0xFF8e9299),
    isDarkMode: false,
    label: 'SILVER',
  ),
  4: QuacTheme(
    headerGradient: LinearGradient(colors: [Color(0xFFecc66a), Color(0xFFc89d3c), Color(0xFF8a6a1e)]),
    accentColor: Color(0xFFc89d3c),
    isDarkMode: false,
    label: 'GOLD',
  ),
  5: QuacTheme(
    headerGradient: LinearGradient(colors: [Color(0xFF2c2c2c), Color(0xFF0a0a0a)]),
    accentColor: Color(0xFFd8dce0),
    isDarkMode: true,  // dark mode completo en Black
    label: 'BLACK',
  ),
};

// Uso:
final theme = statusThemes[clienteStatus.categoria.categoria] ?? statusThemes[1]!;

3. Aplicar el tema en el header del Home

El cabezote del Home (donde dice "Hola Francisco") usa el headerGradient del tema en lugar del morado actual fijo.

Para BLACK (isDarkMode = true):

  • Cambiar el Scaffold.backgroundColor del Home a #1a1a1d
  • Cambiar textos a blanco/plateado
  • Cards también con fondo oscuro

4. Mostrar un badge pequeño con el nombre del nivel

En algún lugar visible del Home (ej. esquina superior del cabezote, debajo del nombre del usuario), mostrar:

text
[PLUS] / [SILVER] / [GOLD] / [BLACK]

Con tipografía bold + el color de acento del tema. Para QUAC (nivel base) no mostrar badge.

5. (Opcional pero recomendado) Mostrar puntos actuales

Si hay espacio, debajo del badge:

text
3.500 Quacs

Donde "Quacs" es el nombre que decidimos para los puntos.

Fuera del Plan B mínimo (NO implementar en legacy)

  • ❌ Pantalla "Mi Nivel" completa con barra de progreso + checkpoints + logo Quac moviéndose (eso queda solo en v2)
  • ❌ Pantalla "Comparar niveles" (queda en v2)
  • ❌ Sección "Promociones para ti" con mini-cards (queda en v2)
  • ❌ Tab "Nivel" en bottom navigation (la legacy queda con sus 5 tabs originales)
  • ❌ Pantalla detalle "Quacs por vencer" con historial mes a mes (queda en v2)
  • ❌ Modal de "subiste de nivel" con animación (queda en v2)
  • ❌ Aplicar el descuento del nivel en el checkout — esto YA lo hace api-compras-quac del lado del backend cuando el cliente paga, no requiere cambio en el mobile legacy

Manejo de errores

El endpoint /StatusQuac/cliente/{nit} falla

Si el cliente es nuevo o el CRM no tiene su registro:

  • HTTP 404 / 500
  • Fallback: asumir categoria.categoria = 1 (QUAC base) y aplicar el tema morado original
  • La app sigue funcionando, simplemente sin tema de color especial

El endpoint responde lento (timeout)

  • Timeout sugerido: 3 segundos (no más)
  • Si tarda más → mostrar el tema base (QUAC nivel 1) y dejar que el endpoint termine en background
  • Cuando llegue la respuesta, refrescar el tema si el cliente tiene un nivel distinto

El nit no se puede convertir a int

  • Validar antes de llamar al endpoint
  • Si el nit es inválido → no llamar, tema base

Caching

Recomendación: cachear el response del cliente por 5 minutos en memoria.

  • Cliente abre app → 1 llamada → guarda en memoria con TTL 5 min
  • Cliente navega entre pantallas → reusa caché
  • Si pasan 5 min y abre Home otra vez → nueva llamada
  • Si app se mata y se vuelve a abrir → nueva llamada

Para datos más estables (catálogos /categorias y /beneficios):

  • Cachear por 24 horas o por sesión
  • Estos casi nunca cambian, no vale la pena pedirlos cada vez

Pruebas para QA

Caso Cómo verificar
Cliente nivel QUAC (base) Login con NIT que tenga categoria=1 → header morado original
Cliente nivel PLUS NIT con categoria=2 → header gradiente vinotinto-morado + badge "PLUS"
Cliente nivel SILVER NIT con categoria=3 → header gradiente plateado + badge "SILVER"
Cliente nivel GOLD NIT con categoria=4 → header gradiente dorado + badge "GOLD"
Cliente nivel BLACK NIT con categoria=5 → Home entero en dark mode + acento plateado
Cliente sin registro QUAC NIT que dispare 404 en el CRM → app muestra tema base (QUAC) sin error visible
Sin conexión a internet Apagar wifi → app muestra tema base + Home funciona

Para validar con el dev del legacy

  • ¿Tu app ya tiene un ThemeProvider o themeNotifier central, o el tema está hardcodeado por pantalla?
  • ¿Vas a inyectar el categoria en el provider del Home (cupoInfo o donde sea), o crear un provider nuevo statusQuacProvider?
  • ¿El cabezote del Home está en un widget único o duplicado en múltiples pantallas? (si está duplicado, hay que cambiarlo en cada una)
  • ¿Hay alguna pantalla del bottom nav que también tenga cabezote y deba ser tematizada? (probable: Movimientos, Mi perfil)
  • ¿El dark mode de BLACK afecta también las pantallas internas del Home (cartera detalle, planes de pago) o solo el cabezote?
  • ¿Si el endpoint del CRM tarda en responder, el Home se queda esperando o se pinta primero con tema base?
