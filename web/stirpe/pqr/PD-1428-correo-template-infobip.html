PD-1428 · Template Infobip

Plantilla con placeholders para cargar en Infobip

Este es el HTML exacto que se sube a Infobip (o cualquier proveedor SMTP/SES con templates). Los {{placeholders}} los reemplaza el motor de envío con los datos reales del PQR antes de enviar el correo.

1 · Variables del template

Las 3 variables que envía get-api al momento de disparar el correo.

Variable
Tipo
Ejemplo / Descripción
{{nombreCliente}}
string
"Juan David"
Primer nombre del cliente. Usado en el saludo "Hola {{nombreCliente}},"
{{numeroTicket}}
int / str
28547
ID del incidente retornado por el ERP CRM. Aparece en el asunto y en la card central.
{{urlConsulta}}
url
https://stirpe.co/pqrs/consulta?ticket=28547
URL completa al detalle del PQR con el ticket precargado en el query string.
Asunto del correo (también con variable):
Confirmación de PQRS · Ticket #{{numeroTicket}}

{{correoCliente}}

6 · Cómo dispara get-api el correo

Ejemplo del payload que PqrStirpeService envía al endpoint de Infobip (Task A6 del plan).

POST https://api.infobip.com/email/3/send
Authorization: App {INFOBIP_API_KEY}
Content-Type: multipart/form-data

{
  "messages": [
    {
      "templateId": 42,
      "from": "Stirpe <notificaciones@stirpe.co>",
      "to": "juan.perez@correo.com",
      "subject": "Confirmación de PQRS · Ticket #28547",
      "placeholders": {
        "nombreCliente": "Juan David",
        "numeroTicket": 28547,
        "urlConsulta": "https://stirpe.co/pqrs/consulta?ticket=28547"
      }
    }
  ]
}
